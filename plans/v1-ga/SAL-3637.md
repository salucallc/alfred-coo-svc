# SAL-3637: Cockpit activity stream SSE even endpoint

## Target paths
- src/co_w3_a_cockpit_activity_stream_sse_even.py

## Acceptance criteria
* Every tool call from any sub generates an SSE frame within 2s of execution
* Cockpit shows live as it happens; filterable by event_type
* Pause / resume works
* Reconnect logic (EventSource auto-reconnect on disconnect)
* Backpressure: server caps to 100 events/sec/client; older drops

## Verification approach
- Run the FastAPI app locally, connect with an SSE client, verify events are received within 2 seconds.
- Test filtering by the ``event_type`` query parameter.
- Simulate pause/resume via the ``/pause`` and ``/resume`` control endpoints.
- Verify reconnection handling by dropping the client and reconnecting.
- Ensure the server does not exceed 100 events/sec per client (assert count).

## Risks
- Potential high CPU usage under load.
- Edge cases in back‑pressure throttling may drop events unexpectedly.
