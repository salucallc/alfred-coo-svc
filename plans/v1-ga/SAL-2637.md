# SAL-2637: Infisical service + pg schema

## Target paths
- deploy/appliance/infisical/init.sql
- deploy/appliance/infisical/README.md

## Acceptance criteria
- `curl infisical:8080/api/status` returns ok; psql shows `infisical` schema with >5 tables

## Verification approach
1. Deploy the appliance with the updated compose configuration.
2. Run `curl http://infisical:8080/api/status` inside the network; expect a JSON response containing `"status":"ok"`.
3. Connect to the `mc-postgres` container and execute `\dt infisical.*`; verify that at least five tables are listed.

## Risks
- Schema incompatibility if the Infisical version changes.
- Service may fail to start if the database credentials are incorrect.
- Exposing the service without proper auth could leak secrets.
