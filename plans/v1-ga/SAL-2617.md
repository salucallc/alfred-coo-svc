# SAL-2617: Endpoint memory push loop

## Target paths
- src/alfred_coo/fleet_endpoint/__init__.py
- src/alfred_coo/fleet_endpoint/memory_push.py
- tests/test_fleet_memory_push.py

## Acceptance criteria
- `1000 writes reflected in hub fleet_memory_sync_log within 60s`
- `local_seq` monotonic
- dup push idempotent

## Verification approach
- Unit and integration tests exercise the push loop, asserting the log count and sequence monotonicity.
- Load test simulates 1000 writes and checks hub side for timely reflection.

## Risks
- Potential overload of hub write path under high write rates.
- Idempotency handling must correctly dedupe duplicate pushes.
