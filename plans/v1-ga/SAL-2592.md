# SAL-2592: Split Docker Networks for COO Isolation

## Target paths
- `deploy/appliance/docker-compose.yml`
- `deploy/appliance/tiresias/network_split.md`

## Acceptance criteria
- `docker exec alfred-coo curl --max-time 5 https://api.github.com` fails with DNS/conn-refused
- `docker exec mcp-github curl https://api.github.com` succeeds

## Verification approach
- Deploy the updated compose file locally.
- Execute the two curl commands above and confirm the expected outcomes.
- Run CI tests (`ruff` + `pytest`) to ensure no regressions.

## Risks
- Potential service disruption if networks are mis‑assigned.
- Needs careful coordination with other tickets that modify networking.
