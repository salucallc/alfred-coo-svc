# SAL-2592: Network split for TIR-10

## Target paths
- `deploy/appliance/docker-compose.yml`
- `deploy/appliance/tiresias/network_split.md`

## Acceptance criteria
```
docker exec alfred-coo curl --max-time 5 https://api.github.com fails
docker exec mcp-github curl same URL succeeds
```

## Verification approach
- Run the above `docker exec` commands inside the respective containers after deployment.
- Ensure `alfred-coo` cannot resolve or connect to `api.github.com` (timeout or DNS error).
- Verify `mcp-github` can successfully `curl` the same URL and receive a 200 response.

## Risks
- R1: If `internal:true` network setting breaks DNS resolution for internal services; fallback to iptables OUTPUT REJECT in init container as per plan.
- R2: Misconfiguration could allow unintended egress from `alfred-coo`; ensure only `mc-egress` network has internet access.
