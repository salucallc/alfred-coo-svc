# AB-22: Boot consumer keys and verify secret handling

## Target paths
- src/alfred_coo/__init__.py

## Acceptance criteria
- [ ] First boot mints all four consumer keys; second boot is a no-op.
- [ ] File permissions verified 0600 on POSIX; skipped on Windows with warn.
- [ ] `verify()` returns `None` for unknown tokens; returns populated `ConsumerIdentity` for known.
- [ ] Unit tests cover: mint, reload, verify-hit, verify-miss, idempotent re-boot.
- [ ] Secret is not stored on disk (only hash baked into key string).

## Verification approach
Added unit tests exercising consumer key minting, reload, verification hit/miss, and ensuring the second boot is a no‑op.

## Risks
- Potential regression if future changes affect boot sequence; tests guard against this.
