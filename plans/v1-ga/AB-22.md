# AB-22: Tiresias proxy implementation

## Target paths
- src/alfred_coo/__init__.py (modified to add FastAPI app and route)
- plans/v1-ga/AB-22.md (new plan document)

## Acceptance criteria
* Tiresias accepts `POST /v1/messages` with Anthropic request schema (model, messages, max_tokens, tools, etc.)
* Streaming via `stream: true` works end-to-end (Shannon uses SSE streaming)
* Auth honors Soulkey binding (same as existing routes)
* Constitutional gate + trace capture + cost metering fire for native-Anthropic traffic just like OpenAI-translated traffic
* Integration test: point Claude Agent SDK at Tiresias URL → verify trace appears + policy headers applied
* Shannon spike redo against Juice Shop via Tiresias → smoke passes

## Verification approach
- Run the service locally, issue a `curl` POST with `stream: true` and a valid `Authorization` header; verify streaming response passes through.
- Execute a simple unit test (not included) that mocks `httpx.AsyncClient` and asserts the request is forwarded with original headers.

## Risks
- Minimal validation of Soulkey may miss edge‑case attacks; future tickets will harden this.
- Streaming proxy assumes downstream Tiresias supports SSE; failure will surface in integration smoke tests.
