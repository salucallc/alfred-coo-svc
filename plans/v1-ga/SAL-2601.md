# SAL-2601: Model Router Implementation

## Target paths
- aletheia/app/router/__init__.py
- aletheia/app/router/policy.py
- aletheia/tests/test_router.py

## Acceptance criteria
- The router must return the correct verifier model for each (action_class, risk_tier) pair.
- The router must raise a ValueError when the selected verifier model equals the generator model.

## Verification approach
- Run `pytest aletheia/tests/test_router.py`; all tests must pass.
- Ensure no lint errors (`ruff`).

## Risks
- Incorrect routing may cause verifier to block legitimate actions.
- Raising exception could surface as internal error; callers must handle.
