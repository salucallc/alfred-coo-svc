# SAL-3636: Live Activity Panel – All View

## Target paths
- src/co_w2_a_cockpit_live_activity_panel_all_.py

## Acceptance criteria
* Panel renders 5+ rows when >=5 subs active
* Each row shows truncated session_id, current_task, age, last_heartbeat, node_id, harness
* Updates every 5s (poll) or live (SSE)
* Empty state renders cleanly when no active subs
* Filter chips work (subs / orchestrators / all)

## Verification approach
- Run Cockpit UI locally.
- Verify empty placeholder when no activities.
- Populate `LiveActivityPanel._activities` with ≥5 mock entries and confirm five rows render.
- Check each column displays correct truncated ID and age.
- Observe updates on periodic `update` calls.
- Use filter chips to narrow results by subsystem/orchestrator/all.

## Risks
- Current implementation only provides a stub; integration with real SSE/polling may require additional schema changes.
- Truncation assumes `session_id` longer than 8 characters.
- Timezone handling assumes ISO‑8601 timestamps without offsets.
