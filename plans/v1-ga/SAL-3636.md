# SAL-3636: Live activity panel implementation

## Target paths
- `src/co_w2_a_cockpit_live_activity_panel_all_.py` (new file)
- `tests/test_co_w2_a_cockpit_live_activity_panel_all_.py` (new file)

## Acceptance criteria
* Panel renders 5+ rows when >=5 subs active
* Each row shows truncated session_id, current_task, age, last_heartbeat, node_id, harness
* Updates every 5s (poll) or live (SSE)
* Empty state renders cleanly when no active subs
* Filter chips work (subs / orchestrators / all)

## Verification approach
- Unit tests in `tests/test_co_w2_a_cockpit_live_activity_panel_all_.py` cover rendering, filtering, empty state, and poll update behaviour.
- Manual smoke test: instantiate `LiveActivityPanel` in a REPL, call `render_rows()` and observe correct output; ensure `sse_generator` yields updated rows.

## Risks
- None identified; implementation is self‑contained and does not introduce external dependencies.
