# SAL-2618: Endpoint memory pull loop

## Target paths
- src/alfred_coo/fleet_endpoint/__init__.py
- src/alfred_coo/fleet_endpoint/memory_pull.py (new)
- tests/test_fleet_memory_pull.py (new)

## Acceptance criteria
- Two endpoints A, B; A writes `shared.*`; B's local search returns within 90s

## Verification approach
- Unit test `tests/test_fleet_memory_pull.py` validates the pull function returns the expected structure.
- Integration test in F21 ensures memory written by endpoint A is searchable by endpoint B within 90 seconds.
- `ruff` and `pytest` must pass in CI.

## Risks
- Network latency may delay pull beyond 90s; mitigated by retry/backoff.
- Authentication failure if API key missing; ensure env vars are set during deployment.
