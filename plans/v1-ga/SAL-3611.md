- plugins/saluca-plugin-echo-inbound/pyproject.toml

## Acceptance criteria
**Acceptance:**

1. `pip install saluca-plugin-echo-inbound==0.1.0` works.
2. `saluca-plugin-sdk` test runner executes contract suite against echo plugin and reports pass.
3. A deliberately-broken inbound plugin (e.g. dispatch_inbound returns wrong type) FAILS contract suite with a clear error.

## Verification approach
- Run `pip install .` for the plugin and ensure installation succeeds.
- Execute `pytest -m saluca_plugin_contract` in the repository; the suite should pass for the echo plugin and fail for the broken fixture, confirming contract enforcement.
- Manually register the plugin in a test alfred-coo instance, dispatch a synthetic task, and verify the echoed result appears in the audit feed.

## Risks
- Deviation from SDK contract definitions may cause false positives/negatives; tests guard against this.
- Adding new files increases repository size minimally; no backward‑compatibility impact on existing plugins.
