# SAL-3638: Extend /v1/cockpit/state with subagents array

## Target paths
- `src/co_w2_b_extend_v1_cockpit_state_with_act.py` (new file)

## Acceptance criteria
- `/v1/cockpit/state` response includes `subagents` array
- Each entry has all fields above (session_id, current_task, age, last_heartbeat, node_id, harness)
- Filter excludes sessions older than 5min and non-`agent-*` sessions
- Cap at 50 entries
- No N+1 queries; one mesh_sessions read

## Verification approach
- Function `_fetch_subagent_activity()` queries mesh_sessions once and applies all filters in-memory
- Age calculation uses datetime parsing and comparison
- Session_id filtering uses string prefix matching
- Result capping happens during iteration (early exit at 50)
- Integration with existing cockpit_router.py structure preserved

## Risks
- datetime parsing failures could skip valid sessions (mitigated: defensive parsing with fallback)
- Timezone handling: all timestamps in UTC (consistent with existing codebase)
- Empty results: subagents array will be empty if no matching sessions (acceptable)
