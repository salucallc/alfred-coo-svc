# SAL-2628: Endpoint docker-compose + bootstrap README

## Target paths
- deploy/endpoint/docker-compose.yml
- deploy/endpoint/.env.template
- deploy/endpoint/README.md

## Acceptance criteria
- Clean Debian VM can be prepared in <10 minutes.
- Operator pastes the `registration_token` into `.env.template`.
- Running `docker compose up -d` registers the endpoint with the hub and receives periodic heartbeats.
- Heartbeat logs show `mode_state=normal` and successful registration.

## Verification approach
- Manual test on a fresh Debian VM following the README.
- Capture container logs; assert presence of `201 Created` registration response.
- Verify that heartbeats appear at the configured interval (default 15 s) for at least two cycles.

## Risks
- Network outbound restrictions could block WSS; fallback to HTTP long‑poll is provided.
- Missing registration token leads to `400 token_invalid` – ensure the README emphasises token placement.
- Docker version incompatibility on older Debian; recommend Docker Engine 26+.
