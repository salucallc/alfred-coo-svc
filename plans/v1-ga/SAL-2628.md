# SAL-2628: Endpoint docker-compose and bootstrap

## Target paths
- deploy/endpoint/docker-compose.yml
- deploy/endpoint/.env.template
- deploy/endpoint/README.md
- deploy/endpoint/bootstrap.sh

## Acceptance criteria
- Endpoint docker-compose + bootstrap README | Fresh operator registers endpoint end-to-end <10min on clean Debian

## Verification approach
- On a fresh Debian VM, copy the files, set `REGISTRATION_TOKEN` in `.env`, run `bootstrap.sh`, and confirm the services start (`docker compose ps` shows both containers up) and the endpoint registers with the hub (observe heartbeat logs). Entire process must complete within 10 minutes.
- Automated CI test spawns a Docker-in-Docker container, runs the bootstrap, and validates registration via a mock hub endpoint.

## Risks
- Operator may forget to set `REGISTRATION_TOKEN`; bootstrap aborts with clear message.
- Docker Compose version mismatch; using compose v2 syntax to minimize issues.
- Network connectivity to hub required; failure leads to retry logic in service.
