# SAL-2628: Endpoint docker-compose + bootstrap

## Target paths
- `deploy/endpoint/docker-compose.yml`
- `deploy/endpoint/.env.template`
- `deploy/endpoint/README.md`
- `deploy/endpoint/bootstrap.sh`

## Acceptance criteria
- Fresh operator registers endpoint end-to-end <10min on clean Debian

## Verification approach
- Run `bootstrap.sh` on a fresh Debian VM after inserting a registration token into `.env`.
- Verify Docker Compose starts the `alfred-coo-endpoint` service.
- Confirm logs contain successful registration (`POST /v1/fleet/register` 201) and periodic heartbeat messages.
- Ensure total elapsed time from script start to first heartbeat is under 10 minutes.

## Risks
- Operator may forget to replace placeholder token in `.env`; mitigated by clear README instructions.
- Docker installation may fail on non‑standard Debian variants; bootstrap checks and aborts with error messages.
- Network connectivity to hub required; if blocked, registration will timeout (handled by script).
