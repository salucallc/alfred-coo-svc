# SAL-2600: Verify prompt template + sentinel parser

## Target paths
- aletheia/app/prompt/__init__.py
- aletheia/app/prompt/parser.py
- aletheia/prompts/verify_v1.md
- aletheia/tests/test_parser.py

## Acceptance criteria
- Parser against 20 canned outputs (10 well‑formed, 10 malformed). All 10 well‑formed → correct `(verdict, rationale)`; all 10 malformed → `ParseError`. `pytest tests/test_parser.py` green. Prompt file sha256 pinned in env.

## Verification approach
- Run `pytest aletheia/tests/test_parser.py` and ensure all tests pass.
- Run `ruff check aletheia/app/prompt/` to confirm lint compliance.
- Manual sanity‑check: invoke `PromptParser.parse` on sample outputs and verify the returned verdict matches the sentinel.

## Risks
- Parser may incorrectly match similar patterns; unit tests cover typical cases.
- SHA256 pinning of the prompt file must be kept in sync with future changes.
