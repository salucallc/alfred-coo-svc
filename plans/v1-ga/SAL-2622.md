# SAL-2622: Emergency policy push (immediate)

## Target paths
- src/mcctl/commands/policy.py
- tests/test_mcctl_policy_immediate.py
- plans/v1-ga/SAL-2622.md

## Acceptance criteria
- `mcctl policy push --immediate` interrupts endpoint <5s; in-flight task requeued with `requeue_reason=policy_immediate`

## Verification approach
- Unit tests in `tests/test_mcctl_policy_immediate.py` verify the CLI flag is passed to the MC client.
- Integration test (existing CI) confirms that when the endpoint receives an immediate policy push, logs show an interrupt within 5 seconds and any running task is re‑queued with the correct `requeue_reason`.

## Risks
- Ensure the immediate flag propagates correctly to the fleet service; otherwise endpoints may not interrupt.
- Added code must not affect existing `mcctl policy` commands for non‑immediate pushes.
- Deleting or overwriting existing files is avoided; only new files are added.
