# F14: Emergency policy push (immediate)

## Target paths
- src/mcctl/commands/policy.py
- tests/test_mcctl_policy_immediate.py

## Acceptance criteria
- `mcctl policy push --immediate` interrupts endpoint <5s
- in-flight task requeued with `requeue_reason=policy_immediate`

## Verification approach
- Unit test `tests/test_mcctl_policy_immediate.py` asserts that `push(immediate=True)` returns an `interrupted` flag, the correct `requeue_reason`, and completes in under 5 seconds.
- Integration tests (not in this ticket) will exercise the command via the full `mcctl` CLI.

## Risks
- The placeholder implementation only simulates the behavior; production code must integrate with the endpoint runtime to actually trigger interruption and requeue.
- Timing assumptions may vary on heavily loaded systems; the 5‑second bound should be revisited in performance testing.
